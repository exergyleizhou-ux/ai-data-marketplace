# 部署手册(从零到上线)

> 面向境内生产部署。代码侧已就绪;本手册覆盖**需要你/运维**操作的部分。涉及付费、实名、备案的步骤只有账号持有人能做。

## 0. 总览

两条路径,二选一:
- **单机(docker compose)**:一台云主机,运维最省,适合起步 → [`deploy/docker-compose.prod.yml`](../deploy/docker-compose.prod.yml)。
- **Kubernetes**:多实例/自动扩缩/滚动发布 → [`deploy/k8s/`](../deploy/k8s/)。

不论哪条,**数据库/Redis/对象存储都用云托管服务**(不自建容器):
- Postgres 16 → RDS(阿里云 RDS / 腾讯 TencentDB)
- Redis → 云数据库 Redis(限流 + H4 刷新令牌 denylist 跨实例共享)
- 对象存储 → OSS / COS(直接用现有 S3 驱动,改环境变量即可)

## 1. 前置(**最先做,周期最长**)

1. **云账号 + 实名认证**。
2. **域名**:购买 + 实名。
3. ⚠️ **ICP 备案**:境内强制,约 1–3 周;经营性业务可能需经营性 ICP。**今天就提交,别等。**
4. 公司主体资质(营业执照、对公账户)——支付分账线也要用(见 [`资金分账合规与集成方案.md`](资金分账合规与集成方案.md))。

## 2. 开通托管资源

- RDS Postgres 16:建库 `ai_data_marketplace`,开 SSL,放在与应用同一 **VPC**,只允许内网访问。
- 云 Redis:开密码,同 VPC。
- OSS/COS:建桶 `ai-data-marketplace`,**私有读写**(下载走应用签发的预签名 URL,不要公开桶)。
- 计算:一台 ECS(单机路径)或 ACK/TKE 集群(k8s 路径)。
- 负载均衡 + **HTTPS 证书**(云 LB 托管证书,或 k8s cert-manager)。

## 3. 密钥(进 KMS,绝不进仓库)

```bash
openssl rand -hex 32   # 生成 JWT_SECRET
openssl rand -hex 32   # 生成 PII_SECRET
```

按 [`.env.production.example`](../.env.production.example) 填齐:`DATABASE_URL`(sslmode=require)、`REDIS_URL`、`S3_*`、`JWT_SECRET`、`PII_SECRET`、支付密钥。
- k8s:用 `kubectl create secret`(见 [`k8s/10-config.yaml`](../deploy/k8s/10-config.yaml) 注释)或 external-secrets。
- 单机:放 `deploy/.env`(`chmod 600`,不入库)。
- ⚠️ 生产强制:`KYC_AUTO_APPROVE=false`、`APP_ENV=production`(此时若 `JWT_SECRET` 仍是默认值,进程**拒绝启动**)、`CORS_ALLOW_ORIGIN` 设真实域名(**不要 `*`**)。

## 4. 构建并推送镜像

```bash
# 后端(distroless,产物 /api)
docker build -t $REGISTRY/ai-data-marketplace-backend:$TAG backend/
docker push $REGISTRY/ai-data-marketplace-backend:$TAG

# 前端 ⚠️ NEXT_PUBLIC_* 是构建期注入的,必须在 build 时给真实值
docker build -t $REGISTRY/ai-data-marketplace-frontend:$TAG \
  --build-arg NEXT_PUBLIC_API_BASE_URL=https://app.example.com/api/v1 \
  --build-arg NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_or_test_xxx \
  frontend/
docker push $REGISTRY/ai-data-marketplace-frontend:$TAG
```

> 前端 Dockerfile 已声明这两个 `ARG`(默认空,等同本地构建);生产构建务必通过 `--build-arg` 传真实值,否则客户端会回退到 localhost/sandbox。

## 5. 跑数据库迁移(一次性步骤)

迁移用专门的一次性命令 `api --migrate`(幂等、带锁,可重复跑):

```bash
# 单机
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env run --rm migrate
# k8s
kubectl -n marketplace apply -f deploy/k8s/20-migrate-job.yaml
kubectl -n marketplace wait --for=condition=complete job/db-migrate --timeout=120s
```

服务端 `AUTO_MIGRATE` 保持 `false`,避免多副本滚动时争抢迁移。

## 6. 部署

**单机**:
```bash
docker compose -f deploy/docker-compose.prod.yml --env-file deploy/.env up -d
```

**k8s**(按序):
```bash
kubectl apply -f deploy/k8s/00-namespace.yaml
# 创建 backend-secrets(见 10-config.yaml 注释),再:
kubectl apply -f deploy/k8s/10-config.yaml
kubectl apply -f deploy/k8s/20-migrate-job.yaml && kubectl -n marketplace wait --for=condition=complete job/db-migrate --timeout=120s
kubectl apply -f deploy/k8s/30-backend.yaml -f deploy/k8s/40-frontend.yaml -f deploy/k8s/50-ingress.yaml
```

## 7. 健康检查与监控

- `GET /healthz`(liveness)、`GET /readyz`(含 DB ping)、`GET /metrics`(Prometheus)。
- 容器健康检查用 `/api --healthcheck`(distroless 无 shell/curl)。
- 接 Prometheus 抓 `/metrics`(`marketplace_http_*`、`quality_jobs_*` 指标)。

## 8. 灰度与回滚

- 灰度:先发 1 副本/小流量,验证注册→实名→浏览→下单→支付→交付→确认→分账闭环,再放量。
- 回滚:`kubectl rollout undo deploy/backend -n marketplace`,或 compose 切回上一个 `$TAG`。迁移**不自动回滚**(向后兼容设计;如需回退迁移用 `migrate down` 谨慎操作)。

## 9. 上线前安全 checklist

- [ ] `APP_ENV=production`、`JWT_SECRET`/`PII_SECRET` 为强随机、`KYC_AUTO_APPROVE=false`。
- [ ] `CORS_ALLOW_ORIGIN` = 真实域名(非 `*`)。
- [ ] 全站 HTTPS;对象存储桶私有,仅预签名访问。
- [ ] DB/Redis 仅内网可达,开 SSL/密码。
- [ ] `git grep -nE 'sk_(test|live)_|whsec_|PASSWORD='` 仓库内零命中(占位符除外)。
- [ ] ICP 备案通过、域名解析生效。
- [ ] 监控/告警接好;备份策略(RDS 自动备份)开启。

## 10. 支付

生产默认维持**已验证的 Stripe(测试或正式)**或 sandbox mock。**境内微信/支付宝真实分账是外部墙**,接入前必须完成资质 + 法务 + 持牌方签约,流程见 [`资金分账合规与集成方案.md`](资金分账合规与集成方案.md)。
