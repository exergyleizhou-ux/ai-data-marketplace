# P3 MVP - 用户故事与粗粒度接口列表

**文档目的**：将设计文档中的 P3 模块进一步拆解成可直接用于任务拆分和开发的用户故事 + 关键接口定义。

**依据**：已固化的 MVP 关键决策（2026-05-31 版本）
- 优先支持中文文本 + 代码数据
- P3 数据类型：纯文本（txt/md）+ JSON/CSV
- 定价：平台建议价 + 卖家可调整
- 支付：微信+支付宝+国际，资金托管模式（10% 佣金）
- 质量：基础自动化 + 人工审核

---

## 1. 用户体系模块

### 用户故事

**卖家/买家通用**
- US-1.1 作为访客，我可以注册账号（邮箱/手机号 + 密码）
- US-1.2 作为用户，我可以通过邮箱/手机号登录
- US-1.3 作为用户，我可以完善个人/企业资料
- US-1.4 作为用户，我必须完成实名认证（个人身份证或企业营业执照）才能发布数据集或购买数据

**角色区分**
- US-1.5 作为用户，我可以同时成为卖家和买家
- US-1.6 作为平台运营，我可以查看所有用户列表并进行基础管理

### 关键接口（粗粒度）

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | /api/v1/auth/register | 注册 | 公开 |
| POST | /api/v1/auth/login | 登录，返回 JWT | 公开 |
| GET  | /api/v1/users/me | 获取当前用户信息 | 登录用户 |
| PUT  | /api/v1/users/me | 更新个人/企业资料 | 登录用户 |
| POST | /api/v1/users/me/kyc | 提交实名认证材料 | 登录用户 |
| GET  | /api/v1/admin/users | 后台用户列表 | 运营角色 |

**核心实体**：User, UserProfile, KycRecord

## 2. 数据集上传模块

### 用户故事

- US-2.1 作为卖家，我可以创建数据集草稿（填写名称、描述、领域、许可类型、样本量估算、定价建议）
- US-2.2 作为卖家，我可以分片上传大文件（支持断点续传）
- US-2.3 作为卖家，上传完成后系统自动进行基础质量检测（格式、编码、简单统计）
- US-2.4 作为卖家，我可以在审核通过前修改数据集信息
- US-2.5 作为运营人员，我可以审核卖家提交的数据集（通过/拒绝 + 备注）

### 关键接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | /api/v1/datasets | 创建数据集（元数据） | 卖家（已实名） |
| POST | /api/v1/datasets/{id}/upload/init | 初始化分片上传 | 卖家 |
| PUT  | /api/v1/datasets/{id}/upload/part | 上传分片 | 卖家 |
| POST | /api/v1/datasets/{id}/upload/complete | 完成上传 | 卖家 |
| GET  | /api/v1/datasets/{id} | 获取数据集详情 | 公开/登录 |
| PUT  | /api/v1/datasets/{id} | 更新元数据（审核前） | 卖家 |
| POST | /api/v1/admin/datasets/{id}/review | 审核数据集 | 运营 |

**支持的文件类型（P3）**：.txt, .md, .json, .csv（可配置）

**核心实体**：Dataset, DatasetVersion, UploadTask, QualityCheckResult

---

## 3. 数据集浏览与搜索模块

### 用户故事

- US-3.1 作为买家，我可以浏览所有已上架的数据集列表（支持分页、排序）
- US-3.2 作为买家，我可以按领域、数据类型、价格区间、质量分数筛选
- US-3.3 作为买家，我可以用关键词搜索数据集
- US-3.4 作为买家，我可以查看数据集详情页（描述、样本示例、许可、卖家信息、历史评分）

### 关键接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET  | /api/v1/datasets | 列表 + 筛选 + 搜索 | 公开 |
| GET  | /api/v1/datasets/{id} | 详情页 | 公开 |
| GET  | /api/v1/datasets/{id}/preview | 获取少量样本预览（安全脱敏） | 登录用户 |

**核心实体**：Dataset（公开可见字段）

---

## 4. 交易与订单模块

### 用户故事

- US-4.1 作为买家，我可以对感兴趣的数据集发起购买（选择许可类型、确认最终价格）
- US-4.2 作为买家，我可以看到平台建议价格，并可与卖家议价（初期可简化为卖家最终确认价）
- US-4.3 作为买家，我可以查看我的订单列表和状态
- US-4.4 作为卖家，我可以查看我收到的订单
- US-4.5 作为平台，我需要记录完整的订单状态流转（待支付 → 已支付/托管 → 已交付 → 已确认 → 已结算 / 纠纷中）

### 关键接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | /api/v1/orders | 创建订单 | 买家 |
| GET  | /api/v1/orders | 我的订单列表 | 登录用户 |
| GET  | /api/v1/orders/{id} | 订单详情 | 订单相关方 |
| POST | /api/v1/orders/{id}/confirm-delivery | 买家确认收货（触发结算） | 买家 |
| POST | /api/v1/orders/{id}/dispute | 发起纠纷 | 买家/卖家 |

**订单状态**（建议）：
`created → paid(托管) → delivered → confirmed → settled`
`disputed → resolved`

**核心实体**：Order, OrderItem, LicenseType

---

## 5. 数据交付模块

### 用户故事

- US-5.1 作为买家，订单支付成功后，我可以获取临时下载链接（带有效期）
- US-5.2 作为买家，我下载的数据应带有平台水印或 traceable 标识（防止二次传播）
- US-5.3 作为卖家，我可以知道我的数据集被哪些订单下载过（基础审计）

### 关键接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET  | /api/v1/orders/{id}/download | 获取临时下载链接 | 订单买家（已支付） |
| GET  | /api/v1/files/{token} | 实际下载（通过临时 token） | 带有效 token |

**交付策略（P3 建议）**：
- 生成带时效的预签名 URL（对象存储）
- 可选：下载时在文件头部/尾部注入买家+订单信息（简单水印）

## 6. 支付与分成模块（含资金托管）

### 用户故事

- US-6.1 作为买家，我可以用微信、支付宝、国际信用卡支付订单
- US-6.2 作为平台，买家支付后资金进入平台托管账户
- US-6.3 作为卖家，买家确认收货后，我可以申请提现（扣除 10% 平台佣金）
- US-6.4 作为平台运营，我可以看到所有交易流水和佣金统计

### 关键接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | /api/v1/payments/create | 创建支付订单（返回支付二维码/链接） | 买家 |
| POST | /api/v1/payments/webhook | 支付渠道回调 | 支付网关 |
| POST | /api/v1/orders/{id}/settle | 触发结算（确认后调用） | 系统/运营 |
| GET  | /api/v1/sellers/me/earnings | 卖家收益与可提现金额 | 卖家 |

**资金流（P3）**：
买家支付 → 平台托管账户 → 买家确认 → 扣 10% 后结算给卖家

---

## 7. 基础权限与安全模块（横切）

### 用户故事

- US-7.1 系统必须验证用户登录态（JWT）
- US-7.2 只有实名用户才能上传和购买
- US-7.3 下载链接必须有时效性和访问控制
- US-7.4 所有敏感操作需要记录审计日志

### 关键能力

- JWT + Refresh Token 机制
- RBAC 基础角色（普通用户 / 卖家 / 买家 / 运营 / 管理员）
- 接口权限中间件
- 审计日志（上传、交易、下载、审核）

---

## 8. 后台运营模块（P3 必须包含）

虽然前台是 C 端，但 P3 必须有基础运营能力：

- 人工审核数据集
- 查看用户 KYC 材料
- 处理纠纷订单
- 查看基础交易统计

**建议单独一个 Admin 后端或嵌入现有后端**（推荐后期独立）。

---

## 建议的 P3 交付物优先级排序（推荐）

1. 用户体系 + 实名认证（基础）
2. 数据集上传 + 分片上传 + 基础质量检测
3. 后台人工审核流程
4. 数据集列表与详情
5. 订单创建 + 支付集成（先做一个支付渠道）
6. 资金托管 + 结算逻辑
7. 数据交付（临时链接 + 简单水印）
8. 其他支付渠道 & 完善
