# 生产部署 — 一键自托管栈

把整个平台跑成一个真实、自托管、带 HTTPS 的部署。一台装了 Docker 的服务器即可。

## 需要什么
- 一台 Linux 服务器(2 vCPU / 4GB 起),装了 Docker + Docker Compose v2
- 一个域名,A 记录指向这台服务器(用于自动 HTTPS)
- 开放 80 / 443 端口

> 纯本地试跑(不要域名/HTTPS):把 `DOMAIN` 设为 `localhost`,Caddy 会签发本地证书。

## 三步起飞
```bash
# 1. 配置
cp .env.prod.example .env.prod
$EDITOR .env.prod          # 填 DOMAIN、改掉所有 CHANGE_ME 密钥

# 2. 起栈(首次会构建镜像 + 自动建库表 + 自动建 S3 桶)
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d --build

# 3. 看状态
docker compose -f docker-compose.prod.yml ps
curl -fsS https://$DOMAIN/healthz      # → ok
```
打开 `https://你的域名` 即是上线站点。

## 这套栈是真的(不是 mock)
| 组件 | 实现 |
|---|---|
| HTTPS | Caddy 自动签发 / 续期 Let's Encrypt 证书 |
| 反向代理 | `/api/*`·`/healthz`·`/metrics`·`/docs` → 后端;其余 → 前端,**同域** |
| 对象存储 | MinIO(S3 兼容,桶自动创建)。换成 AWS S3 / 阿里云 OSS / 腾讯 COS 只需改 `S3_ENDPOINT` + 凭证 |
| 数据库 | Postgres 16,命名卷持久化,首启自动迁移 |
| 计算沙箱 | L1 真 Docker 隔离沙箱(`docker run --network none`) |

## 诚实平替(需要真实世界资源那几项)
这些不是代码能补的,用**可插拔 + 诚实标注**顶上,默认安全:

- **支付**:默认 `PAYMENT_PROVIDER=mock`(不动真钱)。设为 `stripe` 并填真 key 即可走 Stripe 真实收款。**中国微信/支付宝分账**需持牌支付机构 + 资金存管——这是法务/牌照门槛,不是配置。保持 `mock` 直到牌照到位。
- **实名(KYC)**:默认 `KYC_AUTO_APPROVE=false`,提交进**人工运营审核队列**(运营后台可批),这是正当的运营模式,替代第三方实名 API。
- **L2 机密计算**:需真 TEE 硬件(Intel TDX / AMD SEV)。无硬件时计算回落到 L1 沙箱;要开 L2 在 TEE 节点上设 `COMPUTE_RUNNER=tee` + `TEE_ATTESTER=tdx`(见 `部署-L2-TEE节点与KBS.md`)。软件版非对称证明可作演示,但生产守卫会拒绝无硬件提供 L2。
- **多方 PSI**:协议 + HTTP 传输已就绪(`psi_transport.go`),真正多运营方部署是把各方进程放到不同主机 + mTLS。

## 运维
```bash
# 日志
docker compose -f docker-compose.prod.yml logs -f backend
# 升级(拉新代码后)
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d --build
# 备份(数据全在命名卷)
docker run --rm -v ai-data-marketplace_pgdata:/v -v "$PWD":/b alpine tar czf /b/pg-backup.tgz -C /v .
```

## 真正上线中国数据交易前还需要(非工程)
数据交易牌照 / 备案、PIPL 合规与数据出境评估、持牌支付通道、等保测评、法务条款审定。这些是资质与合规问题,代码已为它们留好可插拔接口。
