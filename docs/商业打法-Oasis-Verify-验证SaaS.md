# Oasis Verify — 验证 SaaS 商业打法 + 产品规格

> **盈利优先的最短路:** 不先开数据市场(双边冷启动最慢),而是把已建好的"质量+完整性+溯源验证"引擎,做成一个**自助、按量付费的 B2B 验证 API**。一边生意、卖软件、全球 Stripe 收款、无需国内数据交易牌照。市场撮合是后续愿景,从验证客户里长出来。

## 1. 商业打法(Business Playbook)

**产品:** `Oasis Verify` — 自助式「数据集质量 + 溯源验证 API」。上传/指向一个数据集 → 跑完整性+质量筛查 → 出一张**可验证、可分享**的证书(复用现有 PaperGuard 引擎 + 沙箱 + `/verify`)。

**目标客户(按付费意愿):**
1. **ML/AI 团队 & 数据工程师** — 训练前/采购前给数据体检(垃圾进垃圾出代价高)。
2. **数据卖方/供应商** — 要一张可信"质量+版权+溯源"证书,卖高价 / 过采购审查。
3. **研究者 / 期刊 / 合规** — 查第三方数据完整性。

**为什么付(ROI):** 数据质量问题很贵(训练废、论文撤、采购踩雷);一次几块钱的自动体检 + 一张第三方可验证、可分享的证书,ROI 清楚。**证书是差异点** —— 本地能跑检查,但拿不到可分享、可验证的凭证。

**怎么收钱:** 免费档(每月 N 次)→ 自助注册拿 API key → 超量按次 / 订阅($29/$99/$299)。全程 Stripe(订阅 + 计量)。**卖软件,不碰国内二清/数据交易牌照。**

**获客(创始人发帖,代码侧产出素材):**
- 英文落地页 + 零摩擦免费自助档。
- 病毒钩子:可嵌入「Oasis-verified ✓」证书徽章(已有 `/verify/:id/badge.svg`,扩展即可)。
- 内容引流:「我把 HuggingFace 最热的 N 个数据集做了完整性体检,发现了这些」(诚实、可跑、强引流)。
- 渠道:Show HN / r/MachineLearning / X / Product Hunt / 相关 Discord。

**国内 vs 国际:** 变现先用**国际**(付费意愿 + 绕开牌照 + Stripe);国内"数据要素"大平台是更大更慢的后续愿景,用前者的钱养。

**诚实边界:** 代码侧能把"自助收钱的产品 + 上云基础设施"建好;获客/销售是创始人的活。"代码→自动有钱"不存在,但能让收钱在技术上自助、门槛最低。

## 2. 怎么长成"市场愿景"(解决冷启动)

| 阶段 | 做什么 | 收入 |
|---|---|---|
| **1 验证 SaaS** | 自助验证 API + 证书 + 计费 | 第一笔收入,学谁付费 |
| **2 供给自生** | 用证书的卖方 → "挂出你已验证的数据集" | 供给侧从验证客户长出来 |
| **3 市场流动性** | 买方为"已验证数据"而来 → 撮合 | 抽成 |
| **4 高端 + 国内** | 敏感数据 C2D + 真 TEE + 数据要素 | 高客单 + 政策红利 |

市场从验证客户里长出来,而非凭空撮合 —— 这是绕过双边冷启动的关键。

## 3. 产品规格(MVP 切片)

自助验证产品复用现有引擎,但走一条比"市场撮合"更薄的自助路径(无 offer/purchase/seller 机制)。

**切片顺序(每片一个 PR,TDD):**
1. **API-key 体系(本片)** — 自助签发/校验/计量/吊销。所有可计费的东西都从它过。
   - 迁移 `000034_api_keys`:`api_keys`(account 关联、key 前缀 + 哈希存储、tier、用量月计数、吊销)。
   - 模块 `apikey`:密钥生成(`sk_live_…`,只哈希入库,明文仅返回一次)+ 校验 + 按 tier 月配额计量 + 吊销。
2. **自助验证端点** — `POST /v1/verify/screen`(API-key 鉴权)→ 在沙箱跑完整性筛查 → 返回报告 + 注册可公开 `/verify` 的证书。
3. **Stripe 计费** — 免费档 + 订阅/计量(需真实 Stripe key,可 test-mode 先接)。
4. **英文落地页** — 转化导向:产品 / demo / 免费档 / "试一下"。

**Tier 模型(MVP):** `free`(5 次/月,≤5MB)· `pro`(500 次/月,≤100MB)· `scale`(自定义)。

## 4. 安全与诚实

- 密钥**只存哈希**(sha256),明文仅签发时返回一次;前缀用于展示/定位。
- API-key 鉴权 fail-closed;超配额 429;吊销立即生效。
- 证书延续现有诚实口径:保证**溯源/完整性**,不保证正确性/无恶意;质量筛查标的是**异常**而非造假。
