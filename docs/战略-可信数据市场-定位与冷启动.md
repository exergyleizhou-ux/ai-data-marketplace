# 战略锁定:可信数据市场 — 定位、冷启动、变现

> 2026-06 决策记录。创始人把决定权全权交给执行方,这是锁定的方向。一句话:
> **我们是一个「可信数据市场」(模式 B:别人上架卖,我们抽成 + 盖"已验证"戳)。
> 主攻窄垂直 = 科研/科学数据。冷启动靠"验证先行"。先收卖家的验证费当快钱,
> 撮合抽成是第二幕。**

## 1. 我们是什么(锁定)

**模式 B — 可信数据市场。** 卖方来上架卖数据;平台提供「完整性体检 + 可溯源/可复现证书」这层信任,并对成交抽成。护城河不是某个 SaaS,是**平台级的"已验证"戳**——别人(HuggingFace 那种免费乱堆)做不到。

**定位语**:可信科研数据市场 —— 每份数据都经完整性体检 + 可溯源认证,放心用于训练与研究。
(英文:The marketplace for verified research data — every dataset integrity-screened + provenance-certified.)

## 2. 为什么主攻"科研/科学数据"当滩头(而不是泛 AI 数据)

双边市场最难的是冷启动拉卖家。科研数据是创始人**有捷径**的赛道:
- **现成的第一批卖家 + 天然信任**:PaperGuard 是已发表、有真实用户的科研数据完整性工具(JOSS / PyPI / 605 测试)→ 它的用户/社区就是拉第一批数据上架的渠道。别的赛道没有这个捷径。
- **正踩风口**:AI-for-science(生物/材料/气候)+ 科研可复现性危机 + 数据可信,三股力叠加;"可验证的科研数据"既稀缺又值钱。
- **差异化真实**:一个"每份数据都做过完整性体检 + 带可溯源/可复现证书"的市场。窄,但能先热;验证后可扩到更广的 AI 训练数据。

## 3. 冷启动打法:验证先行(verify-first)

```
卖家拿数据来 → 付费做"完整性体检 + 拿已验证证书"   ← 第一笔快钱 + 把数据吸进货架(供给)
              → 一键把"已验证数据"挂出来卖
买家(AI-for-science 公司 / 药企生物 / 期刊 / 研究者)→ 为"放心数据"而来 → 撮合成交
              → 平台抽成                              ← 第二幕的大钱
```

关键:**向卖家收的验证费,同时干三件事——立刻见钱 + 给数据盖别人没有的可信招牌 + 把供给吸进来。** 这就是为什么"验证"必须放最前面:它是 B 的破局点,不是另一门生意。

## 4. 钱的时间线(诚实)

| 阶段 | 收入来源 | 快慢 |
|---|---|---|
| 现在 | 卖家的**验证费 / 上架费**(软件,即时) | 快钱 |
| 第二幕 | 买家成交的**撮合抽成** | 慢,要先有流动性 |
| 高端 | 敏感数据的"可用不可见"C2D 计算分成 | 更晚 |

**别只盯抽成**——那跟"快赚钱"打架。快钱在卖家侧的验证/信任费。

## 5. 产品现状 vs 还缺什么

- ✅ **已建好**:市场(上架/买卖/卖家/订单 H1–H7)、验证 + 证书 + `/verify`、可用不可见 C2D、自助验证 API + 密钥控制台、计费骨架。
- ⏳ **要串的故事**:把"**先验证 → 一键上架 → 带'已验证'标签卖**"连成一条清晰的线;公开面重新框成"可信数据市场"(本 PR 起步:首页 hero 已改)。
- 🧍 **只有创始人能做**:用 PaperGuard 社区/科研圈,**去拉第一批科研数据上架**(产品备好,人要去找)。这是 B 绕不开的创业活。

## 6. 不要做(避坑)

- ❌ 一上来就摆成"什么数据都卖"的大超市(双边市场只在窄垂直先热)。
- ❌ 只靠撮合抽成等买家(没流动性前没钱;先收卖家的验证费)。
- ❌ 把公司缩成单个 SaaS(Verify 是货架首发商品 + 冷启动入口,不是终局)。
- ❌ 对证书 over-claim(它证"完整性 + 溯源",不证"法律 100% 没问题";诚实是信任市场的命)。
