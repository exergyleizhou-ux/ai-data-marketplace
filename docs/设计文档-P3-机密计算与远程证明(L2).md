# P3 · 机密计算(L2)与远程证明 — 设计与第一刀

**目标**:把信任阶梯从 L1(买方不可见、**平台仍可见**)推到 **L2(连平台也不可见)**——数据仅在 **TEE(可信执行环境)** 内解密计算,平台运营方、主机 root、云厂商都看不到明文;并由 **远程证明(remote attestation)** 向买方/卖方证明「**只有约定的算法代码、在真 enclave 内、处理了这份数据**」。

> 与主设计文档 §2(信任阶梯)、§9(L2/L3 路线)一脉相承。本期交付**可验证的第一刀**:证明 plumbing + teeRunner 脚手架 + Mock 证明器(密码学绑定/校验),真实 TEE 执行像 dockerRunner 一样**门控**(需 TEE 硬件/云,本环境无法执行,只验证代码层)。

## 1. 威胁模型升级:L1 → L2

| | L1 数据沙箱 | **L2 机密计算(本期)** |
|---|---|---|
| 买方看原始数据 | 否 | 否 |
| **平台运营方/主机 root 看原始数据** | **能** | **不能**(仅 enclave 内明文) |
| 云厂商/Hypervisor 看 | 能 | 不能(内存加密) |
| 信任根 | 平台审核的算法代码 | **TEE 硬件信任根 + 远程证明** |

L1 的诚实短板(§2)正是「平台仍可见」。L2 用 **TEE**(Intel TDX / AMD SEV-SNP 机密虚机、或 SGX+Gramine enclave)+ **内存加密** + **远程证明** 关上这个口子。

## 2. 端到端流程

```
卖方用自己的密钥加密数据集 ──▶ 平台仅持密文(平台看不到明文)
买方提交 L2 作业
  │
  ▼ 平台在 TEE 节点起 enclave(运行 digest 钉死的算法镜像)
  │   ① enclave 远程证明 → 证明「真 TEE + 正确度量(算法镜像 digest)」
  │   ② 证明通过后,卖方密钥经安全通道把数据密钥释放进 enclave(KBS/密钥代理)
  │   ③ 数据仅在 enclave 内解密、计算
  ▼ 输出经输出闸门(同 L1:大小/DP/泄漏)→ 释放
证明报告(quote)写入 compute_jobs.attestation ──▶ 买方/卖方可独立校验
```

本期落地 **①(证明的产生)+ 报告存储/暴露/校验**;②(卖方密钥按证明释放,KBS)与真实 enclave 运行列为 P3 后续(需密钥代理 + TEE 节点)。

## 3. 本期交付(可验证)

- **数据模型**:`compute_jobs.attestation JSONB`(000010 已建)承载证明报告;`Job.Attestation` 暴露给 API。
- **Attester 抽象**(`runner_tee.go`):`Attest(measurement, jobID, outputSHA) → 报告`、`Verify(报告) → {绑定字段, verified}`。证明**绑定**:算法镜像 digest(度量值)+ 作业 id(新鲜性)+ 输出哈希(完整性)。
- **MockAttester**:用 HMAC-SHA256(固定「TEE 签名密钥」替身)对绑定字段签名,`Verify` 重算比对——**密码学地**演示「绑定 + 防篡改 + 可独立校验」,无需真 TEE。改报告任一字段 → 校验失败。
- **teeRunner**:包裹任意基础 Runner(生产=dockerRunner,测试=MockRunner)+ Attester;运行算法后产出证明,**L2 作业未取得有效证明则不放行**。
- **worker**:作业取得证明 → 存入 `compute_jobs.attestation`。
- **API**:`GET /compute/jobs/:id/attestation`(买方或该数据集卖方可查),返回报告 + 校验结论。
- **装配**:`COMPUTE_RUNNER=tee` 选 teeRunner;真实 TEE 节点用真 Attester(Intel TDX `tdx-quote` / SEV-SNP / SGX DCAP),云证明服务校验。

## 4. 真实 TEE 落地(P3 后续,门控)

- **运行时**:阿里云加密计算 / Azure Confidential VM(SEV-SNP)/ GCP Confidential VM / Intel TDX;或 SGX + Gramine。`COMPUTE_DOCKER_RUNTIME` 已可切 `kata`(机密容器)。
- **真 Attester**:从 enclave 取 quote(`/dev/tdx_guest` 等),度量值=算法镜像 digest;**云厂商/DCAP 证明服务**校验 quote 真伪与度量。
- **密钥释放(KBS)**:证明通过后才把卖方数据密钥释放进 enclave(可用 attestation-based KBS,如 Confidential Containers KBS)。
- **国密**:面向数交所可用 SM2/SM3 替代签名/摘要(§16.5)。

## 5. 诚实标注

商品页对 L2 标注「机密计算 · 远程证明」徽章 + 可校验的证明摘要;**只有证明校验通过**才呈现 L2 承诺,否则降级提示。延续「信号非结论 / 不夸大」。
